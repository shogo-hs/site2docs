# AGENTS

## 目的
このリポジトリに貢献する際の前提知識と期待される作業プロセスを共有し、作業者間の一貫性を高めることを目的とします。

## プロジェクト概要
- Python で実装された静的ナレッジベース生成ツールです。
- 依存関係管理には [`uv`](https://github.com/astral-sh/uv) を必ず使用してください。
- 実装コードは `site2docs/src/site2docs` 配下にまとまっています。

## コーディング方針
1. **言語ポリシー**
   - ドキュメント、コメント、docstring、CLI ヘルプは原則として日本語で記述してください。
2. **構成の意識**
   - 既存のモジュール分割（`config.py`、`rendering.py` など）の責務境界を尊重し、役割が混在しないようにしてください。
   - 新しいユーティリティを追加する場合は、既存モジュールに収まるか検討した上で、必要なら新しいファイルを作成してください。
3. **型ヒント**
   - 公開関数・メソッドには型ヒントを付与してください。
4. **例外処理**
   - 例外は具体的な例外クラスを捕捉し、原因が追いやすいメッセージを付与してください。

## テストと検証
- 変更が Python コードに影響する場合は、最低限 `uv run site2docs --help` が成功することを確認してください。
- ユニットテストを追加した場合は `uv run pytest` を実行してください。
- 実行ログやテスト出力を最終コメントに記載する際は、成功・失敗が判別できるように明示してください。

## ドキュメント整備
- README やリファレンスを更新した場合は、変更内容を日本語で要約し、追加したコマンドの実行方法が再現可能か確認してください。
- 図表やコードブロックを追加する際は、Markdown の構文を統一し、必要に応じてコメントで前提条件を補足してください。

## PR メッセージ
- PR のサマリーには主要な機能追加・修正点を箇条書きで記載してください。
- テストセクションでは実行したコマンドと結果を明示してください。

## ファイルスコープ
- この AGENTS.md の指示はリポジトリ全体に適用されます。
